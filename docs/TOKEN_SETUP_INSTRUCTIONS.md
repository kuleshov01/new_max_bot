# Инструкция по настройке токена для Max Bot

## Проблема
Ваш бот не отвечает, потому что токен в файле `config.py` является недействительным. Тестирование показало, что текущий токен `f9LHodD0cOLbrqZQQB8FJPQRbOXS69PJaSxyy7TinCoXhWHGZPqYagJJPHDDHJotUx_zNsyjKSW6V1CvJ_GI` не проходит проверку.

## Решение
Для решения проблемы необходимо получить правильный токен от платформы Max. Ниже приведены шаги для получения действительного токена:

### Шаг 1: Регистрация бота в Max
1. Зайдите на сайт разработчиков Max: https://dev.max.ru/
2. Авторизуйтесь в системе
3. Создайте новое приложение/бота
4. Заполните все обязательные поля:
   - Название бота
   - Описание
   - Аватар (если требуется)
   - Webhook URL (если планируете использовать webhook)

### Шаг 2: Получение токена
1. После создания приложения найдите раздел "API Keys" или "Токены"
2. Создайте новый токен для вашего бота
3. Скопируйте полученный токен

### Шаг 3: Настройка конфигурации
1. Откройте файл `config.py` в текстовом редакторе
2. Замените значение переменной `TOKEN` на ваш новый токен:

```python
# Bot token from Max API
TOKEN = "ваш_новый_токен_здесь"
```

### Шаг 4: Проверка настроек
1. Убедитесь, что BASE_URL указан правильно:
```python
BASE_URL = "https://platform-api.max.ru"
```

2. Сохраните изменения в файле `config.py`

### Шаг 5: Тестирование
1. Запустите тестовый скрипт:
```bash
python3 test_bot.py
```

2. Если тест пройден успешно, запустите бота:
```bash
python3 bot.py
```

## Дополнительные рекомендации
- Не публикуйте токен в открытых репозиториях
- Храните токен в безопасном месте
- Регулярно обновляйте токен по необходимости
- При использовании на production-серверах рекомендуется использовать переменные окружения вместо хранения токена в файле

## Возможные проблемы
- Если токен получен, но бот все равно не работает, проверьте:
  - Правильно ли зарегистрирован бот в Max
  - Активирован ли аккаунт разработчика
  - Правильно ли указаны права доступа для токена
  - Не истек ли срок действия токена

## Проверка работоспособности
После замены токена и запуска бота, проверьте файл `bot.log` для просмотра логов работы бота. В логах будут отображаться все события, ошибки и успешные операции.

## Дополнительные возможности диагностики
Теперь в боте доступна функция получения информации о самом боте. Вы можете протестировать её с помощью команды:

```bash
python3 test_bot_info.py
```

Этот скрипт проверит, может ли бот получить информацию о себе, используя токен. Это дополнительный способ убедиться, что токен действителен и имеет необходимые права.
